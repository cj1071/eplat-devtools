## 3. 函数模板规范

### 3.1 模板格式（snippets/*.json）

```jsonc
{
  "category": "弹窗函数",
  "icon": "💬",
  "items": [
    {
      "id": "geDialog-open",
      "name": "geDialog.open()",
      "description": "打开平台弹窗组件",
      "tags": ["dialog", "ui"],
      "template": "geDialog.open({\n  title: \"${标题}\",\n  width: \"${宽度}\",\n  url: \"${URL}\",\n  buttons: [\n    { text: '确定', handler: function() { /* TODO */ } },\n    { text: '取消', handler: function() { geDialog.close(); } }\n  ]\n});"
    }
  ]
}
```

> **设计决策**：模板内直接使用 `${占位符}` 中文命名，插入时保持占位符，用户在编辑器中手动替换。无需参数表单、无需编译——所见即所得。

### 3.2 模板引擎

```typescript
// lib/snippet-engine.ts
// 直接返回 template 字符串，保留 ${占位符}
function getSnippetCode(item: SnippetItem): string {
  return item.template;
}
```

## 4. JSON 预览编辑器

用于设置页中预览/编辑导入的模板文件。

**方案**：使用 [react-json-view-lite](https://www.npmjs.com/package/react-json-view-lite)（~3KB gzip，无外部依赖）或 [vanilla-jsoneditor](https://www.npmjs.com/package/vanilla-jsoneditor)（功能更强，~40KB）。

推荐 `vanilla-jsoneditor`：支持树形/文本双模式、搜索、校验，适合模板编辑场景。

## 5. GitHub Actions CI/CD

### 构建产物

| 产物 | 说明 |
|------|------|
| `.zip` | 标准 Chrome 扩展包，可通过 `加载已解压的扩展程序` 安装 |
| `.crx` | 签名包，可双击直接安装（需配置签名密钥） |

### 工作流

```yaml
name: 构建扩展
on:
  push:
    tags: ['v*']
  workflow_dispatch:

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: 'pnpm'
      - run: pnpm install
      - run: pnpm wxt build
      - run: pnpm wxt zip

      # 构建 .crx（使用签名密钥）
      - name: 构建 CRX
        uses: nicolo-ribaudo/build-crx@v2
        with:
          extension-dir: .output/chrome-mv3
          crx-file: .output/eplat-devtools.crx
          private-key: ${{ secrets.CRX_PRIVATE_KEY }}

      - name: 发布 Release
        uses: softprops/action-gh-release@v2
        if: startsWith(github.ref, 'refs/tags/')
        with:
          files: |
            .output/*.zip
            .output/*.crx
```

## 6. Git Worktree 并行开发

```bash
# 创建功能分支 worktree
git worktree add ../eplat-devtools-snippets feat/snippet-library
git worktree add ../eplat-devtools-url feat/url-relative-path

# 各 worktree 独立启动开发服务
cd ../eplat-devtools-snippets && pnpm dev
```

| 分支 | 功能模块 |
|------|---------|
| `main` | 主干，阶段合并 |
| `feat/wxt-migration` | 阶段 1 基础迁移 |
| `feat/snippet-library` | 阶段 3 代码片段库 |
| `feat/url-relative-path` | 阶段 2 URL 处理 |
| `feat/gedialog-support` | 阶段 2 geDialog 支持 |
