## 3. 数据流

### 3.1 缩放操作

```
用户操作 (Alt+=)
  → keyboard.ts:onKeyDown()
    → zoom.ts:increaseZoom()
      → applyInlineFallback()    // CSS 变量 + 内联样式
      → CustomEvent('__dz_script_editor_zoom__')
        → ace-bridge.ts:applyFontSize()  // Main World
      → writeZoom() → chrome.storage.local
```

### 3.2 代码片段插入

```
用户在 Sidepanel 点击「插入」
  → SnippetTab → sendToBackground({ type: 'INSERT_SNIPPET', code })
    → background:handleMessage()
      → chrome.tabs.sendMessage(activeTab, message)
        → content/index.ts:onMessage()
          → CustomEvent('__dz_script_editor_insert__')
            → ace-bridge.ts:editor.insert(code) + addMarker(3s 高亮)
```

### 3.3 悬浮球打开 Sidepanel

```
用户点击悬浮球
  → fab.ts → sendToBackground({ type: 'OPEN_SIDEPANEL' })
    → background → chrome.sidePanel.open({ windowId })
```

## 4. 安全边界

| 环境 | 能力 | 限制 |
|------|------|------|
| Sidepanel (React) | UI 渲染、chrome.runtime API | 无法直接操作页面 DOM |
| Background (SW) | chrome.tabs/sidePanel/storage | 无 DOM 访问 |
| Content (Isolated) | 页面 DOM 读写 | 无法访问 window.ace |
| Ace Bridge (Main) | window.ace、全部页面变量 | 无 chrome.* API |

## 5. AI 助手架构设想（阶段 4）

> 本阶段不实现，但架构上需预留通道和扩展点。

### 5.1 整体设想

AI 助手的目标是：开发者在 Sidepanel 中描述需求 → AI 生成 eplat 平台脚本代码 → 一键插入 Ace Editor。

```
┌─────────────────────────────────────────────────────────────────┐
│  Sidepanel — AI 助手 Tab                                       │
│  ┌───────────────────────────────────────────────────────────┐  │
│  │  💬 对话界面                                               │  │
│  │  ┌─────────────────────────────────────────────────────┐  │  │
│  │  │ 用户: 帮我写一个打开弹窗加载 /user/list 的脚本      │  │  │
│  │  │ AI:   好的，这是基于 geDialog 的实现:              │  │  │
│  │  │       ```js                                        │  │  │
│  │  │       geDialog.open({ ... })                       │  │  │
│  │  │       ```                                          │  │  │
│  │  │                            [复制] [插入编辑器]      │  │  │
│  │  └─────────────────────────────────────────────────────┘  │  │
│  │  ┌──────────────────────────────┐ [发送]                  │  │
│  │  │ 输入你的需求...              │                          │  │
│  │  └──────────────────────────────┘                          │  │
│  └───────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 技术架构

```
Sidepanel (AI Tab)
  │ 用户输入 prompt
  ▼
chrome.runtime.sendMessage({ type: 'AI_CHAT', prompt, context })
  │
  ▼
Background (Service Worker)
  ├── 构造 system prompt（注入平台知识）
  ├── 调用 LLM API（OpenAI / Claude / 自建）
  │   API Key 存储在 chrome.storage.local（加密）
  │   或由用户在设置页配置
  ├── 流式返回（SSE → chrome.runtime.sendMessage）
  └── 响应写入对话历史（indexedDB）
  │
  ▼
Sidepanel（渲染 AI 回复 + 代码块）
  │ 用户点击「插入编辑器」
  ▼
同 3.2 代码片段插入流程
```

### 5.3 平台知识库（RAG）

```
📂 knowledge/
├── 📄 api-reference.md      # eplat 平台 API 文档
├── 📄 component-guide.md    # geDialog / geForm 等组件用法
├── 📄 script-patterns.md    # 常见脚本模式
└── 📄 coding-standards.md   # 编码规范

构建时 → embedding 向量化 → 存入 background IndexedDB
运行时 → 用户 prompt → 检索相关文档片段 → 注入 LLM context
```

### 5.4 当前预留的扩展点

| 位置 | 预留内容 | 文件 |
|------|---------|------|
| Sidepanel | 第 4 个 Tab "AI 助手"（本阶段隐藏） | `App.tsx` 中 TABS 数组可直接追加 |
| Background | AI 消息处理函数 | `messaging.ts` 中 MSG 已预留类型 |
| lib/constants | `MSG.aiChat` / `MSG.aiResponse` | 追加即可 |
| wxt.config.ts | `host_permissions` | AI API 域名需声明 |

### 5.5 API 选型建议

| 方案 | 优势 | 劣势 |
|------|------|------|
| **OpenAI** | 生态成熟、GPT-4o 代码能力强 | 需付费、需翻墙 |
| **Claude** | 长上下文、代码推理能力优秀 | 同上 |
| **自建服务** | 内网部署安全可控 | 需搭建推理服务 |
| **本地模型** | 完全离线、零成本 | 代码质量受限于模型大小 |

推荐：先支持 OpenAI / Claude（用户自行配置 API Key），后续可扩展接入自建服务。

## 6. CRX 构建说明

> [!NOTE]
> WXT 原生不支持 `.crx` 构建。`.crx` 是 Chrome Web Store 签名后的产物，或使用 Chrome 浏览器的 `pack-extension` 功能生成。

### 可行方案

| 方案 | 说明 |
|------|------|
| **`wxt zip`** | 生成 `.zip`，可通过 Chrome `加载已解压的扩展程序` 安装（推荐） |
| **Chrome 内置打包** | `chrome://extensions` → 打包扩展程序 → 生成 `.crx` + `.pem` |
| **GitHub Actions** | CI 中用 `chrome` 命令行 `--pack-extension` 生成 `.crx` |
| **Chrome Web Store** | 上传 `.zip` → 审核后自动签名分发（正式渠道） |

### GitHub Actions 自动化

```yaml
# .github/workflows/build.yml 中新增步骤
- name: 构建 CRX
  run: |
    # 使用 Chrome headless 打包
    google-chrome --pack-extension=.output/chrome-mv3 \
                  --pack-extension-key=extension.pem
    mv .output/chrome-mv3.crx .output/eplat-devtools.crx
```

> 对于企业内部使用，`.zip` 解压加载是最实用的方式。`.crx` 在现代 Chrome 中需要关闭安全检查或通过企业策略分发。
